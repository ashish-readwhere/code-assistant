~

code-assistant

setup guide — from git clone to .vsix in one terminal session

prerequisites
Node.js
>= 20.20.1
npm
bundled with Node
Vite
npm install -g vite
Git
any recent version
setup steps
01
Clone the repository
$ git clone git@github.com:ashish-readwhere/code-assistant.git
$ cd code-assistant
02
Install Vite globally
Required once per machine. Skip if already installed.
$ npm install -g vite
03
Install root dependencies
$ npm install
04
Build workspace packages
Local packages must be compiled before the GUI or extension can resolve them. Build in dependency order.
$cd packages/config-types && npm install && npm run build && cd ../..
$cd packages/llm-info && npm install && npm run build && cd ../..
$cd packages/terminal-security && npm install && npm run build && cd ../..
$cd packages/fetch && npm install && npm run build && cd ../..
$cd packages/config-yaml && npm install && npm run build && cd ../..
$cd packages/openai-adapters && npm install && npm run build && cd ../..
05
Install core dependencies
The GUI imports directly from core/ — its deps must be present.
$ cd core && npm install && cd ..
06
Build the GUI
Compiles the React frontend via Vite. Output is bundled into the extension package.
$cd gui && npm install && npm run build && cd ..
07
Package the extension
Produces the installable .vsix inside extensions/vscode/build/
$cd extensions/vscode
$npm install
$npm run esbuild
$npm run package
# outputs: build/<name>-<version>.vsix
run everything at once — from repo root

install the extension
$ code --install-extension extensions/vscode/build/*.vsix
Or right-click the .vsix file in VS Code Explorer → Install Extension VSIX.
The output filename is controlled by name and version in extensions/vscode/package.json. Update those fields to rename the packaged file.
rename the extension
Update extensions/vscode/package.json
Three fields control the identity of the packaged extension. Edit them before running npm run package:

"name" — sets the .vsix filename → "code-assistant"
"displayName" — shown in VS Code Extensions panel → "Code Assistant"
"publisher" — used by vsce for marketplace publishing → your publisher ID
key scripts — extensions/vscode/
command what it does
npm run esbuild compile extension with source maps (development build)
npm run esbuild-watch compile + watch — rebuilds on every file save
npm run package produce a release-ready .vsix in build/
npm run package:pre-release same, flagged as a pre-release version
npm run tsc:check TypeScript type-check only, no output files
npm run test run unit tests with Vitest
npm run lint ESLint check across src/
npm run lint:fix auto-fix lint errors where possible
active development workflow
# terminal 1 — watch extension source
$ extensions/vscode/ npm run esbuild-watch
# terminal 2 — watch GUI source
$ gui/ npm run dev
# then press F5 in VS Code to launch the extension in debug mode
